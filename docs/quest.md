# 3D 프린터 다중 출력 시 고정비(인건비 포함) 영향 분석

현재 `3d-print-cost-calculator` 저장소의 계산 로직(`script.js`)을 분석한 결과, **"한 번의 출력 작업(1 Batch)"**을 기준으로 원가와 판가를 계산하도록 설계되어 있습니다. 

작은 출력물을 하나의 프린터 베드에 여러 개 올려서 한 번에 출력할 때, 고정비(인건비, 장비 감가상각비 등)에 미치는 영향과 현재 계산기에서의 고려 방식은 다음과 같습니다.

## 1. 현재 계산기(`script.js`)의 한계점 및 분석
현재 계산기에는 **"출력 개수(수량)"**를 입력하는 항목이 없습니다. 
따라서 사용자가 입력한 `인건비(laborCost)`, `디자인비(designCost)`, `마케팅비(marketingCost)` 등의 고정비는 **출력물 1개가 아닌, 베드에 올라간 전체 출력물(1회 출력 작업)에 대한 총비용**으로 합산됩니다.

## 2. 다중 출력 시 고정비에 미치는 실제 영향

### A. 인건비 (Labor Cost) - **가장 큰 절감 효과**
*   **준비 작업:** 슬라이싱, SD카드 이동, 프린터 베드 레벨링, 출력 시작 버튼 누르기 등은 1개를 뽑든 10개를 뽑든 **동일한 시간**이 소요됩니다.
*   **후가공 작업:** 베드에서 떼어내는 작업 시간은 약간 증가하지만, 1개씩 10번 떼어내는 것보다 한 번에 10개를 떼어내는 것이 훨씬 빠릅니다.
*   **결과:** 한 번에 여러 개를 출력하면 **개당 인건비가 획기적으로 감소**합니다. (예: 인건비가 10,000원일 때, 1개 출력 시 개당 10,000원이지만 10개 출력 시 개당 1,000원으로 감소)

### B. 장비 감가상각비 및 전기세 (Machine & Electricity Cost)
*   현재 로직은 `출력 시간(printTime)`에 비례하여 프린터, 노즐, AMS의 감가상각비와 전기세를 계산합니다.
*   1개를 10번 따로 출력하면 매번 베드와 노즐을 가열(Heating)해야 하므로 전기세와 시간이 낭비됩니다.
*   여러 개를 한 번에 출력하면 가열 시간이 1번으로 줄어들고, 헤드 이동 경로가 최적화되어 **전체 출력 시간이 단축**됩니다. 
*   **결과:** `printTime`이 줄어들기 때문에 개당 장비 감가상각비와 전기세도 소폭 감소합니다.

### C. 기타 고정비 (디자인비, 마케팅비 등)
*   디자인 모델링 시간이나 마케팅 비용은 출력 개수와 무관하게 발생하는 매몰 비용 성격이 강합니다.
*   **결과:** 한 번에 많이, 그리고 자주 출력할수록 개당 부담해야 하는 디자인비와 마케팅비가 분산되어 낮아집니다.

## 3. 현재 계산기를 활용하는 방법 (Workaround)
현재 버전의 계산기에서 여러 개를 한 번에 출력할 때의 "개당 단가"를 구하려면 다음 두 가지 방법 중 하나를 사용해야 합니다.

1.  **총합 계산 후 수동 나눗셈 (권장):** 
    *   필라멘트 사용량, 출력 시간, 인건비 등을 **1회 출력(예: 10개 동시 출력) 기준의 총량**으로 입력합니다.
    *   계산된 최종 `판매가`와 `총 원가`를 출력한 개수(10개)로 직접 나누어 개당 단가를 파악합니다.
2.  **입력 시점부터 개당 비용으로 분할 입력:**
    *   인건비가 10,000원이고 10개를 뽑는다면, 계산기의 인건비 입력란에 `1000`원만 입력합니다.
    *   출력 시간과 필라멘트 사용량도 10으로 나눈 값을 입력합니다.

## 4. 향후 개선 제안 (Feature Request)
계산기의 사용성을 높이기 위해 UI와 로직에 **"1회 출력 개수 (Quantity per Batch)"** 입력란을 추가하는 것을 권장합니다.
*   **개선 로직:** `(총 원가 / 출력 개수) = 개당 원가` 및 `(총 판매가 / 출력 개수) = 개당 판매가`를 화면에 추가로 표시해주면 다중 출력 시의 단가 계산이 매우 편리해질 것입니다.
